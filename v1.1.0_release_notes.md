# AV-Unitest v1.1.0 Release Notes

We are excited to announce the release of **AV-Unitest v1.1.0**! This update primarily focuses on execution polish, quality-of-life improvements, and ensuring that our standalone `.exe` runs flawlessly on any environment.

## 🚀 What's New & Improved

### 1. Stealth Execution Engine (No Popups)
We successfully squashed a major usability bug where compiling the tool into a Windowless executable (`.exe`) caused flashing black command prompt boxes whenever a module invoked PowerShell.
* **Fix**: Injected native Windows `CREATE_NO_WINDOW` (`0x08000000`) flags into all ABAE and Atomic Red Team `subprocess` payload launchers.
* **Result**: The entire benchmark suite now runs 100% invisibly in the background. Tests are stealthier, and the UI experience is vastly improved.

### 2. Timezone-Aware Dashboard Synchronization
Previously, tests executed late at night localized incorrectly on the public dashboard (e.g., a test ran at 10 PM would show up as 6 AM the next day).
* **Fix**: Migrated the `results_handler.py` payload generator to output strict **ISO-8601** formatting with the local +08:00 timezone offset (`datetime.now().astimezone().isoformat()`).
* **Result**: The dashboard's JavaScript now parses dates accurately, ensuring your test run times are perfectly synchronized regardless of where you are in the world.

### 3. Public API Key Hardening
Fixed a connection bug that occurred when running the `.exe` on external, non-development machines.
* **Fix**: The executable now falls back to the official public PythonAnywhere upload API key rather than a dummy placeholder.
* **Result**: You can now seamlessly distribute `AV-Unitest.exe` to friends, colleagues, and testers without them experiencing `403 Unauthorized` errors when their results attempt to sync to the global dashboard.

### 4. PyInstaller Module Discovery (`_MEIPASS` Integration)
Resolved an issue where `AV-Unitest.exe` would falsely report "0 Modules Found" upon launch.
* **Fix**: The `module_manager.py` was refactored to dynamically resolve the `sys._MEIPASS` temporary directory when running as a frozen executable.
* **Result**: Built-in modules are correctly bundled and discovered, while the new `modules_config.json` system still allows for external module integration.
