# AV-Unitest v1.2.0 Release Notes

I am excited to announce the release of **AV-Unitest v1.2.0**! This update primarily focuses on execution polish, strict control for data integrity, and fixing pop-up issues. 

## Key Improvements

1. **Custom Module Strict Control**: To ensure data integrity on the public dashboard, I've implemented a strict control measure. If AV-Unitest detects any custom modules (not part of the core 4 modules), the "Upload to Server" button will be automatically disabled. You will still be able to run these tests locally and export the results to `.txt`.

2. **GoPhish Pop-up Fixes**: I noticed flashing PowerShell/cmd windows during the Level 2 and Level 3 phases of the GoPhish testing module. I have refactored the process execution commands to run entirely silently, effectively fixing these popups.

3. **Version Consistency**: I have updated the benchmark tool's UI and the server dashboard to correctly display `v1.2.0`.

4. **Update Notification Bug Fix**: I fixed an issue where running a local future version (like `v1.2.0`) while the server advertised an older version (`1.0.0`) would erroneously display the "update available" banner. The version checker now accurately parses and compares version numbers.

5. **External Module Discovery Fix**: Fixed a bug where external modules packaged alongside the `.exe` failed to discover the core base module due to PyInstaller pathing issues. Custom external modules will now correctly load and display in the GUI.

6. **UI Scalability Fix**: Implemented a scrollable frame for the test modules list on the deployment screen. The application now elegantly handles 5+ loaded modules without incorrectly pushing the "Start Benchmark" button off the screen.

## Updating

If you are running the standalone `.exe`, the auto-update banner should appear upon opening the application. You can click the download link provided to fetch the latest `v1.2.0` executable.
